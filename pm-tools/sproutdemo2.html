Core Concepts

Objects & Messages

[ ] is the universal construct in Sprout v2 — it represents objects, blocks, lists, and callables all at once. There is exactly one composite type.

  • Message-send by juxtaposition: obj msg sends message msg to obj
  • The colon : creates fields on an object
  • One operation (message-send) + one construct (object) = the entire language
sprout

          

        


        
        

          
No Variables — Only Fields

          

            Sprout has no variables in the traditional sense. Values exist only as named fields on objects.
            Name resolution follows a lookup chain through enclosing scopes.
          

          
    
            
  • x : 42 — creates a field, not a variable
    • 
            
  • self is just an ordinary field
    • 
            
  • [] is empty = none
    • 
            
  • Lookup chain: self → mixins → lexical → prelude → host
    • 
          

          

            
sprout

            

          

        


        
        

          
Blocks & Functions

          

            Blocks are the only callable construct. Parameters go before the pipe
            |,
            body after. Supply fewer arguments to curry.
          

          
    
            
  • [ a, b | a + b ] — params before |, body after
    • 
            
  • [ @ + 1 ] — @ is single-param sugar
    • 
            
  • Currying: supply fewer args = partial application
    • 
            
  • [ | "ready!" say ] — zero-arg block
    • 
          

          

            
sprout

            

          

        


        
        

          
Control Flow is Library

          

            There are no built-in if,
            while, or
            for keywords.
            All control flow is implemented via objects and message sends in the standard library.
          

          
    
            
  • Boolean selection: bool [trueVal, falseVal] — true picks first, false picks second
    • 
            
  • and / or short-circuit via laziness
    • 
            
  • Loops are built on continuations, not special syntax
    • 
            
  • The kernel has zero control flow — pure message passing
    • 
          

          

            
sprout

            

          

        


        
        

          
Continuations as Data

          

            A continuation is a pair: (object, current step). Call, return, loop, and resume are all
            continuation manipulation. Continuations are first-class, multi-shot, and persistable.
          

          
    
            
  • A continuation = (object, current step) — serializable data
    • 
            
  • Call/return/loop/resume all manipulate continuations uniformly
    • 
            
  • First-class, multi-shot: capture, replay, fork execution
    • 
            
  • No function coloring — transparent suspension anywhere
    • 
            
  • Persist a continuation to SQLite; resume it later or on a different machine
    • 
          

          

            
sprout

            

          

        


        
        

          
Persistent by Default

          

            Objects survive restarts automatically. The runtime snapshots the object graph to SQLite
            with content-addressed storage and version history. Ephemeral is opt-in.
          

          
    
            
  • Persistent by default; mark ephemeral to opt out
    • 
            
  • Objects survive restarts, crashes, and migrations
    • 
            
  • Backed by SQLite with full version history
    • 
            
  • Content-addressed storage via universal hashing
    • 
            
  • No ORM, no serialization — the runtime is the database
    • 
          

          

            
sprout

            

          

        


        
        

          
Mutability & Copying

          

            Sprout defaults to mutable + deep-copy (value semantics with mutation).
            Opt into immutability for sharing, interning, and permanent hashing.
            COW (copy-on-write) makes copies cheap under the hood.
          

          
    
            
  • Mutable + deep-copy by default — value semantics you can mutate
    • 
            
  • immutable opt-in unlocks sharing, interning, permanent hash
    • 
            
  • COW implementation: copies are free until you write
    • 
            
  • Structural equality via universal hashing
    •

Types & Dispatch

          

            Sprout is duck-typed: send a message, the object answers (or doesn't). There is no class hierarchy.
            Behavior composition uses mixins, not inheritance.
          

          
    
            
  • Duck-typed: send a message, object answers
    • 
            
  • type field convention for introspection
    • 
            
  • Behavior via mixins, not inheritance — flat composition
    • 
            
  • Per-type extensible primitives
    •

      05
      
Design Principles

      

    

    

      

        🌐
        
Run Anywhere

        
One source, two targets. Write once, compile to LuaJIT for raw speed on the server or to modern JavaScript for the browser and edge. No code changes, no #ifdefs, no platform quirks.

      

      

        📚
        
Everything is Library

        
Tiny kernel, no built-in control flow. if, while, for — all library objects. The kernel does exactly one thing: send a message to an object. Everything else is built on top.

      

      

        📌
        
No Variables

        
Only fields on objects. Every name is a field lookup on the current scope object. This unifies local state, object properties, and module exports into a single mechanism.

      

      

        ⏯
        
Continuations as Data

        
Inspect, persist, resume. A continuation is just (object, step) — plain data you can serialize to disk, send over the network, or fork into parallel branches. No function coloring, no async/await divide.

      

      

        📋
        
Value Semantics

        
Deep-copy by default, COW underneath. Every object is yours to mutate freely. No aliasing surprises, no defensive cloning. When you want sharing, opt in with immutable.
