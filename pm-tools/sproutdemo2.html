A variable-free, field-only, message-passing language that compiles to LuaJIT and modern JS.

Core Concepts

Objects & Messages

[ ] is the universal construct in Sprout v2 — it represents objects, blocks, lists, and callables all at once. There is exactly one composite type.

  • Message-send by juxtaposition: obj msg sends message msg to obj
  • The colon : creates fields on an object
  • One operation (message-send) + one construct (object) = the entire language
sprout

          

        


        
        

          
No Variables — Only Fields

          

            Sprout has no variables in the traditional sense. Values exist only as named fields on objects.
            Name resolution follows a lookup chain through enclosing scopes.
          

          
    
            
  • x : 42 — creates a field, not a variable
    • 
            
  • self is just an ordinary field
    • 
            
  • [] is empty = none
    • 
            
  • Lookup chain: self → mixins → lexical → prelude → host
    • 
          

          

            
sprout

            

          

        


        
        

          
Blocks & Functions

          

            Blocks are the only callable construct. Parameters go before the pipe
            |,
            body after. Supply fewer arguments to curry.
          

          
    
            
  • [ a, b | a + b ] — params before |, body after
    • 
            
  • [ @ + 1 ] — @ is single-param sugar
    • 
            
  • Currying: supply fewer args = partial application
    • 
            
  • [ | "ready!" say ] — zero-arg block
    • 
          

          

            
sprout

            

          

        


        
        

          
Control Flow is Library

          

            There are no built-in if,
            while, or
            for keywords.
            All control flow is implemented via objects and message sends in the standard library.
          

          
    
            
  • Boolean selection: bool [trueVal, falseVal] — true picks first, false picks second
    • 
            
  • and / or short-circuit via laziness
    • 
            
  • Loops are built on continuations, not special syntax
    • 
            
  • The kernel has zero control flow — pure message passing
    • 
          

          

            
sprout

            

          

        


        
        

          
Continuations as Data

          

            A continuation is a pair: (object, current step). Call, return, loop, and resume are all
            continuation manipulation. Continuations are first-class, multi-shot, and persistable.
          

          
    
            
  • A continuation = (object, current step) — serializable data
    • 
            
  • Call/return/loop/resume all manipulate continuations uniformly
    • 
            
  • First-class, multi-shot: capture, replay, fork execution
    • 
            
  • No function coloring — transparent suspension anywhere
    • 
            
  • Persist a continuation to SQLite; resume it later or on a different machine
    • 
          

          

            
sprout

            

          

        


        
        

          
Persistent by Default

          

            Objects survive restarts automatically. The runtime snapshots the object graph to SQLite
            with content-addressed storage and version history. Ephemeral is opt-in.
          

          
    
            
  • Persistent by default; mark ephemeral to opt out
    • 
            
  • Objects survive restarts, crashes, and migrations
    • 
            
  • Backed by SQLite with full version history
    • 
            
  • Content-addressed storage via universal hashing
    • 
            
  • No ORM, no serialization — the runtime is the database
    • 
          

          

            
sprout

            

          

        


        
        

          
Mutability & Copying

          

            Sprout defaults to mutable + deep-copy (value semantics with mutation).
            Opt into immutability for sharing, interning, and permanent hashing.
            COW (copy-on-write) makes copies cheap under the hood.
          

          
    
            
  • Mutable + deep-copy by default — value semantics you can mutate
    • 
            
  • immutable opt-in unlocks sharing, interning, permanent hash
    • 
            
  • COW implementation: copies are free until you write
    • 
            
  • Structural equality via universal hashing
    • 
          

          

            
sprout

            

          

        


        
        

          
Types & Dispatch

          

            Sprout is duck-typed: send a message, the object answers (or doesn't). There is no class hierarchy.
            Behavior composition uses mixins, not inheritance.
          

          
    
            
  • Duck-typed: send a message, object answers
    • 
            
  • type field convention for introspection
    • 
            
  • Behavior via mixins, not inheritance — flat composition
    • 
            
  • Per-type extensible primitives
    • 
          

          

            
sprout

            

          

        


        
        

          
Error Handling — Results as Values

          

            Sprout has no try/catch. Instead, operations
            return ordinary objects. Success returns the value directly; failure returns an object with an
            error field. The absence of
            error is success — nothing extra is stored.
          

          
    
            
  • Success: just the value — no wrapper, no overhead
    • 
            
  • Failure: object with error field describing what went wrong
    • 
            
  • error absent = success (free, nothing stored)
    • 
            
  • Check via message send: result error — returns none if no error
    • 
            
  • No stack unwinding, no hidden control flow, no colored functions
    • 
          

          

            
sprout

            

          

          

            Contrast with try/catch: In JS, errors are invisible in the type system and can
            fly across any function boundary. In Sprout, errors are just data — you see them, check them, pass them on.
          

        


        
        

          
FFI & Host Floor

          

            Sprout’s lookup chain naturally falls through to the host global scope.
            Scope resolution IS the FFI. There is no special import syntax —
            if a name isn’t found in self, mixins, lexical scopes, or prelude, it resolves against the host (JS global / Lua _G).
          

          
    
            
  • Lookup chain: self → mixins → lexical → prelude → host global
    • 
            
  • No import, no require — host names are just there
    • 
            
  • Crossing the boundary: wrap host values in Sprout objects coming in; marshal Sprout objects to host types going out
    • 
            
  • Host throws → Sprout panic (unrecoverable, crashes the continuation)
    • 
            
  • Works identically on JS and LuaJIT targets
    •

JS API Wrappers

      
Host APIs become ordinary objects. The FFI floor (§10) means JS globals are reachable by default — wrappers give them a clean Sprout face.

      

    

    


      

        
DOMDOM Manipulation

        
'Wrap the DOM as Sprout objects'

dom : [
  query : [@ | document querySelector @]
  create : [@ | document createElement @]
  body : document body
]

'Usage'
btn : dom query ".my-btn"
btn textContent : "Click me"
btn addEventListener "click" [@ | "Clicked!" say]

'Create and append'
el : dom create "div"
el innerHTML : "Hello from Sprout"
dom body appendChild el

      


      

        
FetchHTTP / Fetch API

        
'Clean wrapper over fetch'

http : [
  get : [url |
    resp : fetch url
    resp json
  ]
  post : [url, data |
    resp : fetch url [
      method : "POST"
      headers : ["Content-Type" : "application/json"]
      body : data JSON stringify
    ]
    resp json
  ]
]

'Usage — transparent suspension, no await'
users : http get "https://api.example.com/users"
users each [u | (u name) say]

      


      

        
CanvasCanvas 2D Drawing

        
'Canvas wrapper — draw with messages'

canvas : [
  el : document querySelector "canvas"
  ctx : el getContext "2d"

  clear : [| ctx clearRect 0 0 (el width) (el height)]
  fill : [color | ctx fillStyle : color, self]
  rect : [x, y, w, h | ctx fillRect x y w h, self]
  circle : [x, y, r |
    ctx beginPath
    ctx arc x y r 0 (3.14159 * 2)
    ctx fill, self
  ]
]

'Chain calls — each returns self'
canvas clear
  fill "#ff6b35"
  rect 10 10 200 100
  fill "#4ade80"
  circle 300 150 50

      


      

        
StoragelocalStorage

        
'Typed wrapper over localStorage'

store : [
  get : [key | localStorage getItem key]
  set : [key, val | localStorage setItem key val]
  remove : [key | localStorage removeItem key]
  clear : [| localStorage clear]
  keys : [| 0 to (localStorage length) map [@ | localStorage key @]]
]

'Usage'
store set "name" "Alice"
(store get "name") say      'Alice'
store keys each [@ | @ say]

      


      

        
EventsEvent Handling

        
'Events are just messages with blocks'

on : [el, event, handler |
  el addEventListener event handler

btn : document querySelector "#go"
on btn "click" [@ | "Pressed!" say]

'Keyboard — same pattern'
on document "keydown" [e |
  ("Key: " + e key) say

AudioWeb Audio API

        
'Web Audio wrapped as Sprout objects'

audio : [
  ctx : AudioContext new
  play : [freq, dur |
    osc : ctx createOscillator
    gain : ctx createGain
    osc frequency value : freq
    gain gain value : 0.02
    osc connect gain
    gain connect (ctx destination)
    osc start
    osc stop (ctx currentTime + dur)
  ]
]

'Play a melody'
[440, 523, 659, 784] each [f |
  audio play f 0.2
]

Concurrency

      

    

    

      Sprout has no async,
      no await,
      no function coloring. Every block can transparently suspend. The cooperative scheduler
      runs on continuations — suspension is just "don’t resume yet."
      Cancellation is just "don’t resume ever."

      08
      
Performance Tiers

      

    

    

      Sprout gives you a performance ladder. Start dynamic, annotate what matters, drop to native when you need to.
      The compiler uses pure and
      immutable to unlock optimizations —
      together they give JIT-optimal conditions.
    


    

      

        

          Tier 1
          
Dynamic Baseline

        

        

          
Io-like dynamic dispatch. Message-send walks the chain every time. Flexible, introspectable, zero annotations.

          

            
sprout — tier 1

            

          

        

      


      

        

          Tier 2
          
JIT-Optimized

        

        

          
Mark blocks pure and values
          immutable. The compiler skips reification, memoizes results, and inlines aggressively.

          

            
sprout — tier 2

            

          

        

      


      

        

          Tier 3
          
Native FFI

        

        

          
Hot loops drop to C/WASM via manual FFI. Write the kernel in Sprout, the inner loop in C. Zero-copy when possible.

          

            
sprout — tier 3

      11
      
Design Principles

      

    

    

      

        🌐
        
Run Anywhere

        
One source, two targets. Write once, compile to LuaJIT for raw speed on the server or to modern JavaScript for the browser and edge. No code changes, no #ifdefs, no platform quirks.

      

      

        📚
        
Everything is Library

        
Tiny kernel, no built-in control flow. if, while, for — all library objects. The kernel does exactly one thing: send a message to an object. Everything else is built on top.

      

      

        📌
        
No Variables

        
Only fields on objects. Every name is a field lookup on the current scope object. This unifies local state, object properties, and module exports into a single mechanism.

      

      

        ⏯
        
Continuations as Data

        
Inspect, persist, resume. A continuation is just (object, step) — plain data you can serialize to disk, send over the network, or fork into parallel branches. No function coloring, no async/await divide.

      

      

        📋
        
Value Semantics

        
Deep-copy by default, COW underneath. Every object is yours to mutate freely. No aliasing surprises, no defensive cloning. When you want sharing, opt in with immutable.
